生成dummy库的Ruby实现

发表于 2016-12-11 | 分类于 技术 |

实现思路

  • 通过判断每行代码是否为我们生成的cpp文件需要，加上很多限制条件，逐行过滤代码

    #!/usr/bin/env ruby
# encoding: UTF-8

require 'fileutils'
require 'pathname'
require 'tempfile'

ddef dump_file filein, fileout
    file = File.new(fileout, "w")
    #把头文件include进来
    file.print "#include<#{File.basename(filein)}>\n"
    IO.foreach(filein){|line|
        if line.include? "#" or line.include? "typedef" or line.include? "namespace" or line.include? "//"
            file.print "\n"
        elsif line.include? ";"
                file.print line.gsub(";", "{}")
        else
            file.print line 
        end
    }
    file.close
end

fail "Usage: #{$0} header output [soname]" if ARGV.size < 2

tempname = Tempfile.new ['dummy', '.cpp']
dump_file ARGV[0], tempname
libname = ARGV[0].gsub("st_", "lib").gsub(".h", ".so")
#编译命令
system "g++ -I#{File.expand_path("../..",ARGV[1])} tempname -std=c++11 -fPIC \
  -shared -o libname"
fail 'fail to run gcc' unless $?.to_i == 0

存在的问题

  • 以上的代码虽然在一般情况下可用，但是当头文件的编写者在头文件中就给函数定义之后，以上的过滤规则将不再适用。

较好的解决方式

  • 我们的脚本的本质工作其实是将仅声明而未定义的函数加上空的定义，所以我们只需要判断仅声明的函数即可。最终实现代码未在此写出。
